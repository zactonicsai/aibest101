Anthropic AI Fluency Framework

The 4 Ds of AI-Assisted Development

Anthropic's recommended cognitive workflow for integrating AI into technical tasks. Master these four phases to get consistently high-quality results from any AI coding assistant.

D1

Describe

Be explicit and precise about your intent, context, and constraints. AI cannot read your mind — the richer your description, the better the output.

Key elements to describe:
  • Programming language, framework & version
  • What the code should do (inputs/outputs/behavior)
  • Non-functional requirements (performance, security, style guide)
  • Constraints (no external deps, must be thread-safe, etc.)
  • Team conventions and existing patterns
// Example Describe prompt
"I'm building a Spring Boot 3.2 REST API.
Generate a @Service class that queries
PostgreSQL via JPA, handles pagination,
and throws custom exceptions. No Lombok."
D2

Direct

Guide the AI's approach, output format, and reasoning style. Direct = you're the architect; AI is the implementer.

Directing techniques:
  • Specify the output format (class, function, YAML, JSON)
  • Tell it to think step-by-step before writing code
  • Set persona: "Act as a senior Go engineer"
  • Use negative instructions: "Do not use global variables"
  • Chain tasks: "First write tests, then implementation"
// Direct technique
"Act as a DevSecOps engineer.
First, explain the security risks.
Then write a Dockerfile following
CIS Benchmark hardening guidelines.
Output only the Dockerfile."
D3

Dialogue

Treat the AI as a senior peer. Iterate, refine, push back, and ask follow-up questions until the output meets your standards.

Dialogue strategies:
  • "That works, but can you refactor for better testability?"
  • "Explain why you chose that approach vs. X"
  • "What edge cases are you not handling?"
  • "Show me the unit tests for this logic"
  • "What would a security reviewer flag here?"
// Follow-up prompts
"Good. Now add OpenTelemetry tracing.
Keep backward compatibility.
What did you change and why?"
D4

Decide

You own the output. Review, validate, test, and make the final call. AI augments — it doesn't replace — your judgment and accountability.

Decision checklist:
  • ✅ Does it meet functional requirements?
  • ✅ Does it follow team coding standards?
  • ✅ Are there security vulnerabilities?
  • ✅ Is it performant and scalable?
  • ✅ Is it tested and observable?
  • ✅ Do you fully understand what it does?
// Never ship blindly
Always run: unit tests, linting,
SAST scan, peer review
before committing AI-generated code.

Core Principles

AI Coding Best Practices

🧠

Context is King

Always provide your tech stack, version, and architectural context. A prompt with context is 3× more likely to generate usable code than a bare question.

🔍

Verify Everything

AI can hallucinate APIs, introduce subtle bugs, or miss edge cases. Always run tests, linting, and security scans on AI-generated code before merging.

🔄

Iterate in Small Steps

Break large tasks into smaller prompts. Request one function at a time, one file at a time. This gives you checkpoints and keeps context focused.

🛡️

Security-First Prompting

Explicitly request secure defaults: parameterized queries, input validation, least-privilege IAM, no hardcoded secrets. AI won't add these unless you ask.

📚

Test-Driven Prompting

Ask for tests before implementation. "Write JUnit 5 tests for this service, then write the implementation that passes them." TDD improves AI output quality significantly.

📦

Treat AI as a Junior Dev

Review AI code like you'd review a PR from a talented but inexperienced developer. It may miss business logic, organizational patterns, or nuanced security requirements.

✅ DO These Things

  • ▸Paste your existing code for refactoring or review tasks
  • ▸Specify the exact error message when debugging
  • ▸Request explanations for generated code
  • ▸Use AI for boilerplate, CRUD scaffolding, and config generation
  • ▸Ask for multiple approaches: "Show me 3 ways to solve this"
  • ▸Use AI to write commit messages and documentation
  • ▸Ask AI to review your code for security/performance issues

❌ AVOID These Pitfalls

  • ▸Pasting secrets, credentials, or PII in prompts
  • ▸Asking for entire application architectures in one prompt
  • ▸Shipping code you don't understand or haven't reviewed
  • ▸Relying on AI for current CVE or vulnerability knowledge (stale data)
  • ▸Using AI for compliance decisions without expert review
  • ▸Ignoring linter warnings on AI-generated code
  • ▸Treating AI output as ground truth for architecture decisions

Service Layer Chat Prompts

Prompt — Full 4D Flow

"Describe: I need a ProductService class for inventory. It manages Product entities (id, sku, name, quantity, price, category). Direct: Use @Transactional, inject ProductRepository and ProductMapper. Methods: findAll(Pageable), findBySku(String), create(ProductRequest), update(Long, ProductRequest), delete(Long). Throw ResourceNotFoundException if not found. Return ProductResponse DTOs. No Lombok. Add SLF4J logging."

Follow-up · Dialogue

"The update method should use optimistic locking. Add @Version to the entity and handle ObjectOptimisticLockingFailureException. Also add a method to bulk update quantities that accepts a Map<Long, Integer> and is processed in a single transaction."

Sample Output — ProductService

@Service
@RequiredArgsConstructor
@Slf4j
public class ProductService {

    private final ProductRepository repository;
    private final ProductMapper mapper;

    @Transactional(readOnly = true)
    public Page<ProductResponse> findAll(Pageable pageable) {
        log.debug("Fetching products page: {}", pageable);
        return repository.findAll(pageable)
                .map(mapper::toResponse);
    }

    @Transactional(readOnly = true)
    public ProductResponse findBySku(String sku) {
        return repository.findBySku(sku)
                .map(mapper::toResponse)
                .orElseThrow(() -> new ResourceNotFoundException(
                        "Product not found: " + sku));
    }

    @Transactional
    public ProductResponse create(ProductRequest request) {
        log.info("Creating product: {}", request.sku());
        if (repository.existsBySku(request.sku())) {
            throw new DuplicateResourceException(
                "SKU already exists: " + request.sku());
        }
        Product product = mapper.toEntity(request);
        return mapper.toResponse(repository.save(product));
    }

    @Transactional
    public ProductResponse update(Long id, ProductRequest req) {
        Product product = repository.findById(id)
                .orElseThrow(() -> new ResourceNotFoundException(id));
        try {
            mapper.updateEntity(req, product);
            return mapper.toResponse(repository.save(product));
        } catch (ObjectOptimisticLockingFailureException e) {
            throw new ConcurrentModificationException(id);
        }
    }

    @Transactional
    public void bulkUpdateQuantities(Map<Long, Integer> updates) {
        log.info("Bulk updating {} products", updates.size());
        updates.forEach((id, qty) -> repository.findById(id)
                .ifPresent(p -> p.setQuantity(qty)));
    }
}

@ExtendWith(MockitoExtension.class)
class ProductServiceTest {

    @Mock ProductRepository repository;
    @Mock ProductMapper mapper;
    @InjectMocks ProductService service;

    @Test
    @DisplayName("findBySku - returns response when found")
    void findBySku_found() {
        var product = new Product();
        var response = new ProductResponse(1L, "SKU-001", "Widget", 10, 9.99, "Tools");
        when(repository.findBySku("SKU-001"))
                .thenReturn(Optional.of(product));
        when(mapper.toResponse(product)).thenReturn(response);

        var result = service.findBySku("SKU-001");

        assertThat(result.sku()).isEqualTo("SKU-001");
        verify(repository).findBySku("SKU-001");
    }

    @Test
    @DisplayName("findBySku - throws when not found")
    void findBySku_notFound() {
        when(repository.findBySku(any())).thenReturn(Optional.empty());
        assertThatThrownBy(() -> service.findBySku("BAD-SKU"))
                .isInstanceOf(ResourceNotFoundException.class)
                .hasMessageContaining("BAD-SKU");
    }

    @Test
    @DisplayName("create - throws on duplicate SKU")
    void create_duplicateSku() {
        when(repository.existsBySku("DUPE")).thenReturn(true);
        var req = new ProductRequest("DUPE", "Widget", 5, 9.99, "Tools");
        assertThatThrownBy(() -> service.create(req))
                .isInstanceOf(DuplicateResourceException.class);
        verify(repository, never()).save(any());
    }
}

Sample — SecurityConfig

@Configuration
@EnableWebSecurity
@EnableMethodSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(
            HttpSecurity http,
            JwtAuthFilter jwtFilter) throws Exception {
        return http
            .csrf(AbstractHttpConfigurer::disable)
            .sessionManagement(s -> s.sessionCreationPolicy(STATELESS))
            .cors(c -> c.configurationSource(corsSource()))
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/auth/**").permitAll()
                .requestMatchers("/actuator/health").permitAll()
                .requestMatchers("/actuator/**").hasRole("ADMIN")
                .requestMatchers(HttpMethod.GET, "/api/**").hasAnyRole("USER","ADMIN","READONLY")
                .requestMatchers(HttpMethod.POST, "/api/**").hasAnyRole("USER","ADMIN")
                .requestMatchers(HttpMethod.DELETE, "/api/**").hasRole("ADMIN")
                .anyRequest().authenticated())
            .addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class)
            .exceptionHandling(e -> e
                .authenticationEntryPoint(new HttpStatusEntryPoint(UNAUTHORIZED))
                .accessDeniedHandler((req, res, ex) ->
                    res.sendError(HttpServletResponse.SC_FORBIDDEN)))
            .build();
    }

    @Bean
    CorsConfigurationSource corsSource() {
        var config = new CorsConfiguration();
        config.setAllowedOrigins(
            List.of(System.getenv("CORS_ORIGINS").split(",")));
        config.setAllowedMethods(List.of("GET","POST","PUT","DELETE","OPTIONS"));
        config.setAllowedHeaders(List.of("Authorization","Content-Type"));
        config.setMaxAge(3600L);
        var source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/api/**", config);
        return source;
    }
}

Sample — main.go

package main

import (
	"context"
	"net/http"
	"os/signal"
	"syscall"
	"time"

	"github.com/rs/zerolog/log"
	"golang.org/x/sync/errgroup"
)

func main() {
	cfg := config.Load()
	logger := logging.New(cfg.LogLevel)
	db := database.Connect(cfg.Database)
	defer db.Close()

	// Wire dependencies
	orderRepo := repository.NewOrderRepository(db)
	orderSvc  := service.NewOrderService(orderRepo)
	handler   := handler.NewOrderHandler(orderSvc, logger)

	// Routers
	router     := router.New(handler)
	healthRtr  := router.NewHealth()

	apiSrv := &http.Server{
		Addr:         ":" + cfg.Port,
		Handler:      router,
		ReadTimeout:  15 * time.Second,
		WriteTimeout: 30 * time.Second,
		IdleTimeout:  60 * time.Second,
	}
	healthSrv := &http.Server{
		Addr:    ":8081",
		Handler: healthRtr,
	}

	ctx, stop := signal.NotifyContext(
		context.Background(),
		syscall.SIGTERM, syscall.SIGINT,
	)
	defer stop()

	g, gCtx := errgroup.WithContext(ctx)
	g.Go(func() error { return apiSrv.ListenAndServe() })
	g.Go(func() error { return healthSrv.ListenAndServe() })
	g.Go(func() error {
		<-gCtx.Done()
		log.Info().Msg("Shutting down...")
		shutCtx, cancel := context.WithTimeout(
			context.Background(), 30*time.Second)
		defer cancel()
		_ = apiSrv.Shutdown(shutCtx)
		_ = healthSrv.Shutdown(shutCtx)
		return nil
	})
	if err := g.Wait(); err != nil {
		log.Fatal().Err(err).Msg("Server error")
	}
}

Handler Generation

Full 4D Prompt

"Generate an OrderHandler struct with Chi router. Methods: GetOrder(w,r), CreateOrder(w,r), UpdateOrderStatus(w,r), ListOrders(w,r). Decode JSON bodies with validation using go-playground/validator. Return structured JSON errors. Add request ID logging via middleware. Inject OrderService interface (not concrete). Idiomatic Go — no OOP patterns."

Sample — Handler

type OrderHandler struct {
	svc    service.OrderService
	logger zerolog.Logger
}

func NewOrderHandler(svc service.OrderService,
    logger zerolog.Logger) *OrderHandler {
	return &OrderHandler{svc: svc, logger: logger}
}

func (h *OrderHandler) GetOrder(w http.ResponseWriter,
    r *http.Request) {
	id, err := uuid.Parse(chi.URLParam(r, "id"))
	if err != nil {
		respondError(w, http.StatusBadRequest,
            "invalid order id")
		return
	}
	order, err := h.svc.GetOrder(r.Context(), id)
	if errors.Is(err, service.ErrNotFound) {
		respondError(w, http.StatusNotFound,
            "order not found")
		return
	}
	if err != nil {
		h.logger.Error().Err(err).Str("id",
            id.String()).Msg("get order failed")
		respondError(w, http.StatusInternalServerError,
            "internal error")
		return
	}
	respondJSON(w, http.StatusOK, order)
}

func (h *OrderHandler) CreateOrder(w http.ResponseWriter,
    r *http.Request) {
	var req CreateOrderRequest
	if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
		respondError(w, http.StatusBadRequest,
            "invalid request body")
		return
	}
	if err := validate.Struct(req); err != nil {
		respondValidationError(w, err)
		return
	}
	order, err := h.svc.CreateOrder(r.Context(), req)
	if err != nil {
		h.logger.Error().Err(err).Msg("create order failed")
		respondError(w, http.StatusInternalServerError,
            "internal error")
		return
	}
	respondJSON(w, http.StatusCreated, order)
}

func TestGetOrder(t *testing.T) {
	orderID := uuid.New()
	mockOrder := &domain.Order{
		ID: orderID, Status: "PENDING",
	}
	tests := []struct {
		name       string
		urlParam   string
		mockSetup  func(*mocks.OrderService)
		wantStatus int
	}{
		{
			name:     "success",
			urlParam: orderID.String(),
			mockSetup: func(m *mocks.OrderService) {
				m.On("GetOrder", mock.Anything, orderID).
					Return(mockOrder, nil)
			},
			wantStatus: http.StatusOK,
		},
		{
			name:       "invalid uuid",
			urlParam:   "not-a-uuid",
			mockSetup:  func(m *mocks.OrderService) {},
			wantStatus: http.StatusBadRequest,
		},
		{
			name:     "not found",
			urlParam: uuid.New().String(),
			mockSetup: func(m *mocks.OrderService) {
				m.On("GetOrder", mock.Anything, mock.Anything).
					Return(nil, service.ErrNotFound)
			},
			wantStatus: http.StatusNotFound,
		},
	}

	for _, tt := range tests {
		t.Run(tt.name, func(t *testing.T) {
			mockSvc := &mocks.OrderService{}
			tt.mockSetup(mockSvc)
			h := NewOrderHandler(mockSvc, zerolog.Nop())
			rr := httptest.NewRecorder()
			req := httptest.NewRequest(http.MethodGet,
				"/orders/"+tt.urlParam, nil)
			rctx := chi.NewRouteContext()
			rctx.URLParams.Add("id", tt.urlParam)
			req = req.WithContext(context.WithValue(
				req.Context(), chi.RouteCtxKey, rctx))
			h.GetOrder(rr, req)
			assert.Equal(t, tt.wantStatus, rr.Code)
		})
	}
}

Pipeline Prompts

Full Pipeline Request

"Generate a GitHub Actions workflow for a Spring Boot app. Stages: 1) Test (unit + integration with Testcontainers), 2) SAST with Semgrep + dependency scan with Trivy, 3) Build & push Docker image to ECR with SBOM generation, 4) Scan image with Trivy (fail on CRITICAL), 5) Deploy to EKS via Helm with blue/green using Argo Rollouts, 6) Smoke test. Use OIDC for AWS auth (no static creds). Parallel jobs where possible."

Pipeline Flow
Push/PR → Unit Tests ‖ SAST Scan
↓
Build Image → Image Scan → Push ECR
↓
Helm Deploy → Smoke Test → ✅ Live

Sample — GitHub Actions

name: Build, Scan & Deploy

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

permissions:
  contents:      read
  id-token:      write   # OIDC
  packages:      write
  security-events: write

env:
  ECR_REGISTRY: ${{ vars.ECR_REGISTRY }}
  IMAGE_NAME:   inventory-api

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with: { java-version: '21', distribution: 'temurin' }
      - uses: actions/cache@v4
        with:
          path: ~/.m2
          key: ${{ runner.os }}-m2-${{ hashFiles('**/pom.xml') }}
      - run: ./mvnw verify -P integration-tests

  sast:
    runs-on: ubuntu-latest
    needs: test
    steps:
      - uses: actions/checkout@v4
      - name: Semgrep SAST
        uses: semgrep/semgrep-action@v1
        with:
          config: p/owasp-top-ten p/java p/secrets
      - name: Dependency scan
        run: |
          ./mvnw dependency-check:check \
            -DfailBuildOnCVSS=7
      - uses: aquasecurity/trivy-action@master
        with:
          scan-type: fs
          format: sarif
          output: trivy-fs.sarif
      - uses: github/codeql-action/upload-sarif@v3
        with: { sarif_file: trivy-fs.sarif }

  build-push:
    runs-on: ubuntu-latest
    needs: [test, sast]
    outputs:
      image-tag: ${{ steps.meta.outputs.tags }}
      digest:    ${{ steps.build.outputs.digest }}
    steps:
      - uses: actions/checkout@v4
      - name: Configure AWS via OIDC
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ secrets.AWS_ROLE_ARN }}
          aws-region:     us-east-1
      - uses: aws-actions/amazon-ecr-login@v2
      - name: Build & push with SBOM
        id: build
        uses: docker/build-push-action@v5
        with:
          push: true
          tags: ${{ env.ECR_REGISTRY }}/${{ env.IMAGE_NAME }}:${{ github.sha }}
          provenance: true
          sbom: true
          cache-from: type=gha
          cache-to:   type=gha,mode=max

  image-scan:
    runs-on: ubuntu-latest
    needs: build-push
    steps:
      - uses: aquasecurity/trivy-action@master
        with:
          image-ref:  ${{ needs.build-push.outputs.image-tag }}
          format:     sarif
          output:     trivy-image.sarif
          severity:   CRITICAL,HIGH
          exit-code:  1

  deploy:
    runs-on: ubuntu-latest
    needs: image-scan
    if: github.ref == 'refs/heads/main'
    steps:
      - uses: actions/checkout@v4
      - name: Configure AWS via OIDC
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ secrets.AWS_DEPLOY_ROLE_ARN }}
          aws-region:     us-east-1
      - run: aws eks update-kubeconfig --name ${{ vars.EKS_CLUSTER }}
      - name: Helm upgrade
        run: |
          helm upgrade --install inventory-api ./charts/inventory-api \
            --namespace production \
            --set image.tag=${{ github.sha }} \
            --set image.repository=${{ env.ECR_REGISTRY }}/${{ env.IMAGE_NAME }} \
            --wait --timeout=5m \
            --atomic

DevSecOps Security Gates

🚫 BLOCK (Fail Build)
  • ▸ Critical CVEs (CVSS ≥ 9.0) in base image
  • ▸ Secrets detected in code or image layers
  • ▸ SAST Critical findings (SQL injection, RCE)
  • ▸ Running as root in container
  • ▸ DAST Critical/High alerts
⚠️ WARN (PR Comment)
  • ▸ High CVEs (CVSS 7.0–8.9) — must be triaged
  • ▸ Deprecated dependencies
  • ▸ Missing security headers
  • ▸ SAST Medium findings
✅ GATE (Required to Pass)
  • ▸ SBOM generated and signed (cosign)
  • ▸ Image provenance attestation
  • ▸ All unit tests pass ≥ 80% coverage
  • ▸ License compliance check clean
